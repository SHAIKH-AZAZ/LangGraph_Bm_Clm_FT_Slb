─────────────────────────────────────────────────────────────────────────
RING TEXT — STEP-BY-STEP PARSING ALGORITHM
─────────────────────────────────────────────────────────────────────────

Given a RING cell text, follow these steps exactly:

  STEP R1 — Find the DIAMETER:
    The DIAMETER is the first number that appears BEFORE the word TOR.
    Example: "8 TOR ..."    → DIAMETER = 8
    Example: "10 TOR ..."   → DIAMETER = 10

  STEP R2 — Detect FORMAT A or FORMAT B:
    Look at what appears IMMEDIATELY after TOR (ignore spaces):
      • If a NUMBER appears before the first @ sign → FORMAT A
      • If @ appears with NO number in between     → FORMAT B

    HOW TO TELL FORMAT A FROM FORMAT B:
      "8 TOR 15 @ 75 ..."    → there is a number (15) before @  → FORMAT A
      "10 TOR 15 @ 75 ..."   → there is a number (15) before @  → FORMAT A
      "8 TOR 12 @ 75 ..."    → there is a number (12) before @  → FORMAT A
      "8 TOR @ 125 C/C"      → @ comes directly after TOR       → FORMAT B
      "10 TOR @ 125 C/C"     → @ comes directly after TOR       → FORMAT B
      "8 TOR @ 75 C/C"       → @ comes directly after TOR       → FORMAT B

  STEP R3 — Extract ENDCOUNT (FORMAT A only):
    The ENDCOUNT is the number between TOR and the first @ sign.
    "8 TOR 15 @ 75 ..."  → ENDCOUNT = 15
    "8 TOR 12 @ 75 ..."  → ENDCOUNT = 12
    "10 TOR 15 @ 75 ..."  → ENDCOUNT = 15
    "10 TOR 13 @ 75 ..."  → ENDCOUNT = 13
    ENDCOUNT is always a small number like 10, 12, 13, 15, or similar.
    If you think the endcount is 75, 100, 115, 125, or 150 — you are WRONG.
    Those are spacing values, not endcounts.

  STEP R4 — Build dia:
    FORMAT A: dia = "{DIAMETER}-T{ENDCOUNT}"
      DIAMETER=8,  ENDCOUNT=15  → dia = "8-T15"
      DIAMETER=8,  ENDCOUNT=12  → dia = "8-T12"
      DIAMETER=10, ENDCOUNT=15  → dia = "10-T15"
      DIAMETER=10, ENDCOUNT=13  → dia = "10-T13"

    FORMAT B: dia = "T{DIAMETER}"
      DIAMETER=8   → dia = "T8"
      DIAMETER=10  → dia = "T10"

    NEVER put a spacing value (75, 100, 115, 125, 150) into dia.
    NEVER put SETS or LINKS count into dia.
    The dia comes ONLY from DIAMETER and ENDCOUNT.

  STEP R5 — Count ALL @ signs in the RING text:
    Go through the entire RING cell text.
    Every @ sign represents one spacing value.
    You must read the number immediately after EACH @ sign.

  STEP R6 — Extract ALL spacing values:
    For each @ sign found in Step R5:
      Read the number immediately after @ — that is a spacing value.
    Collect all spacing values found.

  STEP R7 — Deduplicate and sort:
    Remove duplicate values. Sort ascending (smallest first). Format as "{VALUE} C/C".

  STEP R8 — Ignore SETS / LINKS / NOS / LOOPS:
    Any occurrence of SETS, LINKS, NOS, LOOPS and the number before it → IGNORE.
    These do NOT belong in dia or spacing.

─────────────────────────────────────────────────────────────────────────
SPACING VALUES — ACCURACY RULES (CRITICAL)
─────────────────────────────────────────────────────────────────────────

The ONLY valid spacing values that appear in this type of drawing are:
  75    (seventy-five)
  100   (one hundred)
  115   (one hundred fifteen)
  125   (one hundred twenty-five)
  150   (one hundred fifty)

These are FIVE DISTINCT numbers. You must never confuse them:

  150 ≠ 100   →  150 is "one-five-zero". 100 is "one-zero-zero". They are different.
  115 ≠ 75    →  115 has three digits. 75 has two digits. They are different.
  115 ≠ 100   →  115 ends in 5. 100 ends in 0. They are different.
  125 ≠ 150   →  125 has a 2 in the middle. 150 has a 5 in the middle. Different.
  125 ≠ 75    →  125 has three digits. 75 has two digits. Different.

RULE: Before writing any spacing value, count its digits and verify each digit:
  75:   2 digits → 7, 5
  100:  3 digits → 1, 0, 0
  115:  3 digits → 1, 1, 5
  125:  3 digits → 1, 2, 5
  150:  3 digits → 1, 5, 0

─────────────────────────────────────────────────────────────────────────
SPACING EXTRACTION — THE TWO-@ RULE (MOST COMMON PATTERN)
─────────────────────────────────────────────────────────────────────────

The most common ring pattern in this drawing type is:
  "{D} TOR {N} @ {X} + @ {Y} + {N} @ {X} C/C"

This pattern has THREE @ signs:
  First  @ → followed by X  (a spacing value)
  Second @ → followed by Y  (a DIFFERENT spacing value)
  Third  @ → followed by X  (SAME as first, therefore a duplicate → skip)

So the result has TWO unique spacing values: X and Y.

EXAMPLES — READ THESE CAREFULLY:

  "8 TOR 15 @ 75 + @ 150 + 15 @ 75 C/C 4 SETS + 1 LINK"
    @1 → 75,  @2 → 150,  @3 → 75 (dup)
    spacing = "75 C/C, 150 C/C"   ← TWO values, not one

  "8 TOR 12 @ 75 + @ 150 + 12 @ 75 C/C 4 SETS"
    @1 → 75,  @2 → 150,  @3 → 75 (dup)
    spacing = "75 C/C, 150 C/C"   ← TWO values

  "10 TOR 15 @ 75 + @ 115 + 15 @ 75 C/C 3 SETS + 2 LINKS"
    @1 → 75,  @2 → 115,  @3 → 75 (dup)
    spacing = "75 C/C, 115 C/C"   ← TWO values

  "10 TOR 15 @ 75 + @ 115 + 15 @ 75 C/C 2 SETS + 1 LINK"
    @1 → 75,  @2 → 115,  @3 → 75 (dup)
    spacing = "75 C/C, 115 C/C"   ← TWO values

  "10 TOR 15 @ 75 + @ 115 + 15 @ 75 C/C 2 SETS + 3 LINKS"
    @1 → 75,  @2 → 115,  @3 → 75 (dup)
    spacing = "75 C/C, 115 C/C"   ← TWO values

  "10 TOR 15 @ 75 + @ 115 + 15 @ 75 C/C 9 SETS + 9 LINKS"
    @1 → 75,  @2 → 115,  @3 → 75 (dup)
    spacing = "75 C/C, 115 C/C"   ← TWO values (ignore 9 SETS, 9 LINKS)

CRITICAL: If you extract only ONE spacing value from a pattern that contains
two DIFFERENT @ values, you have made an error. Go back and re-read.

─────────────────────────────────────────────────────────────────────────
SPACING EXTRACTION — THE ONE-@ RULE
─────────────────────────────────────────────────────────────────────────

Some ring patterns have only ONE unique @ value:

  "8 TOR @ 125 C/C 3 SETS + 3 LINKS"
    Only @ → 125
    spacing = "125 C/C"    ← ONE value (this is FORMAT B)

  "8 TOR @ 75 C/C 4 SETS"
    Only @ → 75
    spacing = "75 C/C"     ← ONE value (this is FORMAT B)

  "10 TOR 15 @ 75 + @ 75 C/C ..."
    @1 → 75, @2 → 75 (dup)
    spacing = "75 C/C"     ← ONE value (both @ signs give same value)

─────────────────────────────────────────────────────────────────────────
ANTI-CONFUSION WARNINGS FOR STIRRUPS
─────────────────────────────────────────────────────────────────────────

WARNING 1 — Never read reinforcement text as ring dia:
  If the VERT. REINF. sub-row says "6-25 TOR", the "6" is a bar count.
  It does NOT mean the ring diameter is 6.
  The ring diameter comes ONLY from the RING sub-row.

WARNING 2 — Never mistake endcount for spacing:
  In "8 TOR 15 @ 75", the number 15 is the endcount (hook count).
  It goes into dia as "8-T15". It does NOT go into spacing.

WARNING 3 — Never mistake spacing for endcount:
  In "8 TOR 15 @ 75", the number 75 is the spacing.
  It goes into spacing as "75 C/C". It does NOT go into dia.

WARNING 4 — SETS and LINKS go nowhere:
  In "4 SETS + 1 LINK" or "3 SETS + 3 LINKS":
  The numbers 4, 1, 3, 9 are set/link counts.
  They do NOT go into dia. They do NOT go into spacing. Ignore them completely.

WARNING 5 — Format B cells do NOT have an endcount:
  If the ring text is "8 TOR @ 125 C/C", there is NO endcount.
  dia = "T8"  (NOT "8-T..." anything)
  The @ comes directly after TOR with no number in between.

If RING cell is blank: dia = "",  spacing = ""
