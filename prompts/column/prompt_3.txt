=====================================================
WHAT THIS DRAWING LOOKS LIKE
=====================================================

- The page is a grid of COLUMN CROSS-SECTION DRAWINGS (rectangular boxes with
  rebar dots/circles inside).
- The drawings are arranged in ROWS. Each ROW is ONE FLOOR-LEVEL RANGE, named on
  the LEFT side of the row (for example "FOUNDATION LVL. TO BASEMENT FLOOR LVL.").
- Each COLUMN of drawings belongs to one COLUMN MARK (its ID), written in / near a
  "COLUMN MKD." oval.
- At the BOTTOM there is usually a FOOTING / PEDESTAL summary table (rows such as
  d1, d2, PEDESTAL, R.C.C. SIZE). IGNORE that table — see the IGNORE section.

One cross-section drawing = ONE column entry. Output one record for every
(column mark x floor-level range) cell.

=====================================================
THE TWO KEY FIELDS — DO NOT CONFUSE THEM
=====================================================

column_no   = the COLUMN MARK / ID (read from the "COLUMN MKD." oval or the label
              beside the drawing). Examples of the FORMAT: "C34", "SW1", "TA-C1".
column_name = the FLOOR-LEVEL RANGE of that row (read from the LEFT side).
              Example of the FORMAT: "FOUNDATION LVL. TO BASEMENT FLOOR LVL.".

- Never put a level name in column_no, and never put a column ID in column_name.
- Do NOT use size-type codes like "R.C.C SIZE - S1" / "S2" as column_no — those
  are size references, not column marks.
