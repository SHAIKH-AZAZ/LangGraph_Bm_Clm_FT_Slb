========================================
STEP 1 — LOCATE THE FOOTING TABLE
========================================

CRITICAL: Before extracting ANY data, you must first FIND the footing table.

How to identify it:
- Look for a section header containing the word "FOOTING" (e.g. FOOTING DETAILS, FOOTING SCHEDULE).
- The header area may also show global values like MIX and SBC (e.g. "MIX : M200", "SBC = 35T/SQ.M").
- Directly below the header, there will be rows labeled: Size, D TO d, Short Span, Long Span.
- Below those rows is the COLUMN MARK row with actual column IDs (C1, C2, C3, etc.).

How to confirm you found the RIGHT table:
- The COLUMN MARK row has values like C1, C2, C3... NOT "4TH FLOOR COLUMN" or "BASEMENT COLUMN".
- The Size row has values like "3200 X 4100" (footing dimensions), NOT column cross-section sizes.
- The Short Span / Long Span rows have values like "φ 16 @ 130C/C" (footing reinforcement).

========================================
STEP 2 — READ COLUMN BY COLUMN
========================================

CRITICAL RULE: This table MUST be read COLUMN BY COLUMN. Never read row-wise.

Use the tools in this exact sequence:
1. think — scan the full drawing, locate the FOOTING DETAILS table, and plan extraction.
2. zoom_region — zoom into the FOOTING DETAILS table area first. Then zoom into individual cells for unclear values.
3. confirm_read — record exact text from each zoomed region before using it as evidence.
4. add_footing — submit extracted footing data with source_region_ids.

Rules:
- think must be first and must return structured observable extraction planning data.
- zoom_region must FIRST target the FOOTING DETAILS table area (wherever it appears on the sheet). Then zoom into column mark cells, unclear dimensions, reinforcement, and global notes within that table.
- confirm_read must record exact text from each zoomed region before that region is used as source evidence.
- add_footing must include source_region_ids that support the footing values.
- Do not invent values. If a value is not visible, use null or [].

========================================
STEP 3 — COLUMN LOCKING
========================================

- Locate the row named COLUMN MARK or COL. MARK (e.g. C1,C18 | C2,C9 | C3 ...).
- These define EXACT vertical columns. Create one footing object PER column mark group.
- For each column mark, read UPWARD through the footing section rows (Size, D TO d, Short Span, Long Span) that fall in the same vertical column.
- You MUST NOT shift values between columns.
- If a column has no values in the footing section, all properties must remain NULL.

========================================
STEP 4 — EXTRACTION RULES
========================================

SIZE:
- From the "Size" row in FOOTING DETAILS section.
- Format like "3200 X 4100" → width = first number (3200), length = second number (4100).
- Ignore PCC sizes completely.

DEPTH:
- From the "D TO d" row.
- For "1100 TO 300" → depth = 300 (smaller value).
- For "750 SOLID" → depth = 750.

REINFORCEMENT:
- Two rows — "Short Span" and "Long Span".
- Format like "φ 16 @ 130C/C" → dia = "16", spacing = "130 C/C".
- Extract dia and spacing separately for each span direction.

MIX:
- Extract from the FOOTING DETAILS section header (e.g. "MIX : M200" → "M200").
- If column is REFER PLAN → mix = null.

STEEL GRADE:
- Extract only if visible in the table or global notes, otherwise null.

COLUMN IDS:
- Must be copied exactly as written (e.g. "C1,C18" stays as one string, not split).
- Keep grouped IDs as one string when they share one footing specification.
