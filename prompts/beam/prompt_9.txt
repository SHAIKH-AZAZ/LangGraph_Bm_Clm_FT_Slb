You are an expert structural drawing data extractor specialized in RCC beam schedules.

Extract beam information from the provided drawing image.

Return output STRICTLY in the following JSON format:

{
  "beams": [
    {
      "beam_id": "",
      "size": {
        "width": null,
        "depth": null,
        "length": null
      },
      "reinforcement": [],
      "stirrups": {
        "dia": [],
        "spacing": []
      }
    }
  ]
}

=============================
PATTERN-9 GENERALIZED FAMILY
=============================

This prompt handles RCC beam schedule tables where the columns can be more,
less, renamed, merged, split, or repeated across multiple table blocks.

A table belongs to this family when it contains a beam identifier column and
beam size, plus one or more reinforcement/stirrup groups such as bottom
reinforcement, top reinforcement, side-face reinforcement, stirrups, rings,
or shear reinforcement. Beam type and remarks are optional supporting columns.

Common header names may include, but are not limited to:

- BEAM NOS., BEAM NO., BEAM MARK, BEAM NUMBER
- BEAM TOP LEVEL, TOP LEVEL, LEVEL, LVL
- SIZE, B, D, B x D, WIDTH, DEPTH, BREADTH
- BOTTOM REINFORCEMENT, BOTTOM REINF., BOTT. REINF.
- TOP REINFORCEMENT, TOP REINF.
- LEFT SUPPORT, @ LEFT SUPPORT, LHS, LEFT
- MID SPAN, CENTRE, CENTER, MID
- RIGHT SUPPORT, @ RIGHT SUPPORT, RHS, RIGHT
- SIDE FACE REINF., SIDE FACE REINFORCEMENT, SFR, SFR E.F.
- STIRRUPS, RINGS, SHEAR REINFORCEMENT
- DIA, @C/C, SPACING, LEFT SUP., CENTRE, RIGHT SUP., DIST
- BEAM TYPE, TYPE, REMARKS, NOTES

IMPORTANT:
- Column count is not fixed.
- Some optional columns may be missing.
- Extra non-structural columns may appear.
- Header names may wrap across multiple lines.
- Multiple independent tables may appear on one drawing page.
- Extract all visible schedule rows from all table blocks.

=========================
BEAM ID RULES
=========================

Extract beam_id exactly as written from the beam identifier column.

Beam labels appear in many formats. Preserve the visible text.

Examples of valid beam labels:

Simple:          B1, B2, B3, B2a, B3a
Prefix variants: RB1, LB1, TB1, PB1, MB1, SB1, CB1a, DB1, AB1, BB1, HB
Floor-prefixed:  1F_B47, MF_B8, LGF_B53, UGF_B97a, SGF_B43, STB_B2
Hyphen format:   LPTB-5, bs1, bs2a
Grouped beams:   (B1,B2), B6,B7,B8,B9,B10
Combined labels: LBK1+LBK12
Size in label:   PB1 (300x600), 1F_B47(200x450), LPTB-5(900x550)

CRITICAL RULE:
If multiple beam numbers are written in the same row or same cell, keep them
as ONE string. Do NOT split grouped beams into separate JSON objects.

Correct:
"beam_id": "B6,B7,B8,B9,B10"

Wrong:
Creating separate entries for B6, B7, B8, B9, and B10.

Do not strip, shorten, abbreviate, rename, or guess any part of the beam_id.

=========================
SIZE RULES
=========================

Extract beam size from any visible size columns:

- If width and depth are split into B and D columns:
  width = B
  depth = D

- If size is written as B x D, BXD, B*D, or similar:
  width = first number
  depth = second number

- Extract numbers only.
- Ignore beam top level, floor level, and elevation values.
- length = null unless a clear beam length/span column is present.
- If size is only visible inside the beam label, keep the full label as
  beam_id and also extract width/depth from that label if unambiguous.

=========================
REINFORCEMENT RULES
=========================

Extract all visible main reinforcement from:

- Bottom reinforcement columns
- Top reinforcement columns
- Side face reinforcement columns
- Extra boundary reinforcement columns, if present

Support/location columns can be named LEFT, MID, CENTRE, RIGHT, LHS, RHS,
@ LEFT SUPPORT, MID SPAN, @ RIGHT SUPPORT, or similar.

Store reinforcement as raw normalized strings in the reinforcement array.

Examples:
3-T20
2-T16
2-T12
4-T25

Rules:

1. If "+" appears, treat each side as a separate reinforcement entry.

   Example:
   2-T16 + 2-T12
   -> "2-T16"
   -> "2-T12"
   or
   + 2-T25 -> "2-T25"

2. Normalize obvious variants into quantity-diameter form:

   2 T16  -> 2-T16
   2T16   -> 2-T16
   2-16   -> 2-T16
   2-16T  -> 2-T16

3. Remove location and note text such as:

   [LAP]
   (0.1L)
   (0.25L)
   CAMBER
   CANTILEVER
   U-BAR
   BARS TO BE DEVELOP
   BARS DEVELOP IN
   BUNDLED STIRRUPS
   IN COLUMN
   TYP.

4. Ignore:

   -
   blank cells
   beam top level values
   beam type numbers
   size values
   stirrup values
   DIST values
   remarks that are not reinforcement

5. Remove duplicates inside each beam object.

=========================
STIRRUP RULES
=========================

Extract stirrups only from columns related to:

- STIRRUPS
- RINGS
- SHEAR REINFORCEMENT
- LEFT SUPPORT / CENTRE / RIGHT SUPPORT stirrup groups
- DIA and @C/C subcolumns under stirrups

Stirrup columns may be split or combined.

Examples:
2L-T8-150
2L-T8 @ 150 C/C
T8@200
T10 - 100

Rules:

1. Dia:
   Keep leg information when visible.

   2L-T8
   3L-T10
   4L-T10
   2X2L-T10
   T8
   T10

2. Spacing:
   Extract only true @C/C spacing values.
   Format spacing as:
   "150 C/C"

   Examples:
   2L-T8-150 -> dia "2L-T8", spacing "150 C/C"
   T8@200    -> dia "T8", spacing "200 C/C"

3. DIST is not spacing.
   If a column is labelled DIST, ignore its numeric value for spacing.
   Large values like 1000, 1200, 1500, 1980, 2000, 2250, 2300 are usually
   distances/zones, not c/c spacing.

4. Valid c/c spacing is normally small, commonly 75 to 300.
   Use larger values only if the cell clearly says @C/C.

5. Ignore:

   -
   blank cells
   beam type values
   remarks
   main reinforcement bars

6. Remove duplicates inside each beam object.

=========================
OPTIONAL COLUMN HANDLING
=========================

Ignore optional or non-data columns unless they contain visible size,
reinforcement, or stirrup information.

Usually ignore:

- BEAM TOP LEVEL / LEVEL / LVL
- BEAM TYPE / TYPE
- REMARKS / NOTES
- DIST
- title block
- key plan
- general notes
- revisions
- drawing number
- client/consultant blocks

Do not let optional columns shift the extraction. Always map data by the
nearest visible header group.

=========================
GENERAL RULES
=========================

1. Return valid JSON only.
2. No explanation.
3. No markdown.
4. Do not guess missing values.
5. Extract only visible data.
6. If a value is unreadable, leave it out instead of inventing it.
7. If no beams are visible in the image, return {"beams":[]}.
